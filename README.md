# NewsAgent

NewsAgent 是一个用于**抓取和汇总不同平台热点新闻**的 Agent 项目，目标是帮助你在最短时间内掌握全网正在发生的重要事件。

---

## 项目定位

在信息过载的时代，热点分散在多个平台（门户、社交媒体、资讯聚合、行业站点）。
NewsAgent 通过统一工作流完成：

1. 多平台热点采集
2. 去重与聚合
3. 主题归类
4. 摘要输出

最终产出可直接用于：晨报、选题会、运营看板、舆情追踪等场景。

---

## 核心能力

- **多源热点获取**：面向多个新闻/内容平台拉取实时热门内容。  
- **统一数据结构**：将不同平台数据标准化，便于后续处理。  
- **热点去重与排序**：按热度、时效、平台权重生成优先级列表。  
- **摘要与关键信息提取**：自动提炼标题要点、事件主体与时间线。  
- **可扩展 Agent 流程**：便于增加新平台、新规则和新输出模板。  

---

## 典型使用场景

- **内容运营**：快速生成每日热点选题池。  
- **媒体/编辑团队**：缩短人工检索与信息整理时间。  
- **品牌与公关**：监测行业和竞品相关舆情。  
- **个人信息管理**：构建自己的热点雷达。  

---

## 处理流程（Agent Pipeline）

```text
[热点源采集] -> [字段标准化] -> [去重聚类] -> [热度评估] -> [摘要生成] -> [多格式输出]
```

当前版本已按该流程提供一个最小可运行实现：

- 采集器（Collector）接口 + 2 个示例采集器（门户 / 社交媒体）
- 去重、主题分类、热度评分、摘要提取
- JSON 与 Markdown 两种输出
- 关键词追踪与平台维度榜单

---

## 快速开始

### 1) 运行

```bash
python news_agent.py --top 10 --keywords AI 台风 财报
```

### 2) 参数说明

- `--top`：控制输出的全网热点条数，默认 `10`。
- `--keywords`：可选关键词列表，支持一次传入多个词（空格分隔）。
  - 作用：在标题和正文中匹配关键词，并在报告的「关键词监控」章节单独列出命中结果。
  - 示例：`--keywords AI 台风 财报`。

### 3) 查看输出

默认会生成：

- `output/report.json`
- `output/report.md`

## 设计原则

- **及时性优先**：热点是时间敏感信息。  
- **可解释性**：每条结果尽量保留来源与时间戳。  
- **可扩展性**：平台接入和规则迭代应低成本。  
- **低耦合**：采集、处理、输出模块解耦，便于替换。
